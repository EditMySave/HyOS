# Security Policy

> **Pre-1.0 Disclaimer:** HyOS is pre-release software under active development. The server manager should **not** be exposed to the public internet until the 1.0.0 release. It is designed for use on trusted local networks or behind a VPN.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.x     | :white_check_mark: (best-effort) |

Security patches are applied to the latest release only. Once HyOS reaches 1.0.0, a formal support matrix will be published.

## Reporting a Vulnerability

If you discover a security vulnerability in HyOS, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. Email the maintainers or use [GitHub Security Advisories](https://github.com/editmysave/hyOS/security/advisories) to report privately
3. Include a detailed description of the vulnerability, steps to reproduce, and any potential impact
4. Allow reasonable time for a fix before public disclosure

We take security seriously and will acknowledge reports within 72 hours.

## Security Documentation

For comprehensive security documentation including authentication details, production hardening checklists, and known limitations, see the [Security section](https://editmysave.github.io/hyOS/docs/security) of the HyOS documentation.
